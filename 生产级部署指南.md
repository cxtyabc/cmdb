## 🚀 五步标准部署流程

### 第一步：环境优化 (5分钟)

#### 1.1 内存优化 - ⚠️ 关键步骤
```bash
# 检查当前内存状态
free -h

# 如果可用内存小于1GB，必须创建交换文件
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 验证内存改善
free -h  # 应显示2GB swap
swapon -s  # 确认swap已启用
```

#### 1.2 Docker环境配置
```bash
# 配置Docker镜像加速（国内必需）
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://docker.1panel.live"],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
EOF

# 重启Docker服务
sudo systemctl daemon-reload
sudo systemctl restart docker
sudo systemctl enable docker

# 验证Docker状态
docker --version
docker info | grep "Server Version"
```

#### 1.3 部署路径标准化
```bash
# 创建统一部署目录
sudo mkdir -p /gaodun/domin
cd /gaodun/domin

# 下载项目（如果还没有）
git clone https://github.com/cxtyabc/cmdb.git veops-cmdb
cd veops-cmdb
```

### 第二步：配置文件准备 (10分钟)

#### 2.1 环境变量配置
```bash
# 创建.env文件
cat > .env << 'EOF'
MYSQL_ROOT_PASSWORD=your_secure_password_here
MYSQL_DATABASE=cmdb
MYSQL_USER=cmdb
MYSQL_PASSWORD=123456
EOF

# ⚠️ 生产环境建议修改密码
```

#### 2.2 Docker Compose配置
```bash
# 使用提供的标准docker-compose.yml
# ⚠️ 重要：确保包含以下配置
# - MySQL环境变量
# - Redis环境变量
# - LDAP DNS映射
# - 统一网络配置
```

### 第三步：LDAP配置 (15分钟) ⭐ 关键步骤

#### 3.1 确认LDAP信息
```bash
# 1. 使用ldapsearch查询实际用户DN结构
ldapsearch -x \
  -H ldap://your-ldap-server:389 \
  -b "dc=your-domain,dc=com" \
  "(uid=your-username)" dn

# 输出示例：
# dn: cn=username,ou=Devops,ou=Development,dc=your-domain,dc=com
```

#### 3.2 Web界面配置
**访问**: http://your-server:8000/#/setting/auth

**标准配置格式**：
```yaml
启用认证: ✅ 开启
服务器地址: ldap://your-ldap-server:389
域: your-domain.com
用户DN: cn={},ou=Devops,ou=Development,dc=your-domain,dc=com  # 注意使用{}占位符
```

**⚠️ 关键注意**：
- 使用`{}`或`{0}`占位符，**不要使用`{username}`**（Web界面会转义）
- 通过"测试连接"功能验证配置
- 测试通过后务必点击"保存"

#### 3.3 LDAP DNS映射（如需要）
```bash
# 编辑docker-compose.yml，在cmdb-api服务中添加：
extra_hosts:
  - "your-ldap-server.com:172.29.255.244"

# 重启API容器
docker compose restart cmdb-api
```

### 第四步：服务启动 (10分钟)

#### 4.1 分步启动策略
```bash
# 1. 启动数据库和缓存
docker compose up -d cmdb-db cmdb-cache

# 2. 等待数据库健康
echo "等待数据库启动..."
sleep 20

# 3. 启动API服务
docker compose up -d cmdb-api

# 4. 等待API健康
echo "等待API启动..."
sleep 30

# 5. 启动Web界面
docker compose up -d cmdb-ui

# 6. 验证所有容器
docker ps --format 'table {{.Names}}\t{{.Status}}'
```

#### 4.2 服务验证
```bash
# 1. 检查容器健康状态
docker ps --format 'table {{.Names}}\t{{.Status}}\t{{.Ports}}'

# 2. 测试Web界面
curl -s -o /dev/null -w 'HTTP状态码: %{http_code}\n' http://localhost:8000/

# 3. 测试LDAP登录
curl -X POST http://localhost:8000/api/v1/acl/login \
  -H "Content-Type: application/json" \
  -d '{"username":"your-ldap-user","password":"your-password","auth_with_ldap":1}'

# 应该返回：{"token": "...", "username": "..."}
```

### 第五步：生产优化 (10分钟)

#### 5.1 容器资源限制
```yaml
# 在docker-compose.yml中添加：
services:
  cmdb-db:
    deploy:
      resources:
        limits:
          memory: 1G
        reservations:
          memory: 512M
  cmdb-api:
    deploy:
      resources:
        limits:
          memory: 512M
        reservations:
          memory: 256M
```

#### 5.2 日志配置
```bash
# 创建日志目录
mkdir -p /var/log/cmdb

# 配置日志轮转
sudo tee /etc/logrotate.d/cmdb << 'EOF'
/var/log/cmdb/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
    create 0644 root root
}
EOF
```

#### 5.3 监控配置
```bash
# 容器监控脚本
cat > /usr/local/bin/cmdb-monitor.sh << 'EOF'
#!/bin/bash
echo "=== CMDB服务监控 ==="
echo "时间: $(date)"
echo "容器状态:"
docker ps --format 'table {{.Names}}\t{{.Status}}' | grep cmdb
echo "内存使用:"
free -h | grep Mem
echo "磁盘使用:"
df -h | grep -E '(Filesystem|/$)'
EOF

chmod +x /usr/local/bin/cmdb-monitor.sh

# 添加到cron（每小时检查）
(crontab -l 2>/dev/null; echo "0 * * * * /usr/local/bin/cmdb-monitor.sh >> /var/log/cmdb-monitor.log 2>&1") | crontab -
```

## 🔧 故障排查快速参考

### 常见问题快速修复

#### 问题1：容器无法启动
```bash
# 检查1：端口冲突
netstat -tuln | grep 8000
# 解决：修改docker-compose.yml中的端口映射

# 检查2：内存不足
free -h
# 解决：创建swap文件（见第一步1.1）

# 检查3：权限问题
sudo chown -R $USER:$USER /gaodun/domin/veops-cmdb
```

#### 问题2：LDAP登录失败
```bash
# 诊断步骤
# 1. 测试DNS解析
docker exec cmdb-api ping your-ldap-server.com

# 2. 测试LDAP连接
docker exec cmdb-api python << 'EOF'
from ldap3 import Server, Connection
server = Server('your-ldap-server.com')
try:
    conn = Connection(server, 'cn=testuser,ou=users,dc=domain,dc=com', 'password')
    print(f"LDAP连接: {'成功' if conn.bind() else '失败'}")
except Exception as e:
    print(f"LDAP连接失败: {e}")
EOF

# 3. 检查配置格式
# Web界面 → 设置 → 认证 → LDAP
# 确认使用 {} 或 {0} 占位符，不是 {username}
```

#### 问题3：数据库连接错误
```bash
# 检查1：数据库容器状态
docker ps | grep cmdb-db
# 应该显示健康状态

# 检查2：DNS解析
docker exec cmdb-api ping cmdb-db
# 应该能ping通

# 检查3：环境变量
docker exec cmdb-api env | grep MYSQL_HOST
# 应该显示 MYSQL_HOST=cmdb-db

# 修复：重启服务
docker compose restart
```

#### 问题4：Web界面500错误
```bash
# 检查API日志
docker logs cmdb-api --tail 50

# 检查nginx日志
docker logs cmdb-ui --tail 20

# 常见原因：
# - 数据库未就绪：等待30秒后重试
# - 表结构问题：执行 flask common-check-new-columns
# - 权限问题：检查数据卷权限
```

#### 问题5：端口冲突
```bash
# 快速修复：修改端口映射
# 在docker-compose.yml中修改：
ports:
  - "8001:80"  # 将8000改为8001

# 或停止占用端口的进程
sudo lsof -ti:8000 | xargs kill -9
```

## 🛠️ 运维命令参考

### 日常运维
```bash
# 服务管理
docker compose ps                    # 查看状态
docker compose logs -f cmdb-api     # 查看API日志
docker compose restart cmdb-api     # 重启API服务
docker compose up -d                # 启动所有服务
docker compose down                 # 停止所有服务

# 备份
docker exec cmdb-db mysqldump -ucmdb -p123456 cmdb > backup_$(date +%Y%m%d).sql

# 监控
docker stats                        # 实时资源监控
docker system df                    # 磁盘使用
```

### 故障恢复
```bash
# 完全重启（不删除数据）
docker compose down
docker compose up -d

# 重建服务（保留数据卷）
docker compose down
docker volume prune -f
docker compose up -d

# 紧急回滚
git checkout <working-commit>
docker compose down
docker compose up -d --force-recreate
```
