## 📋 部署前准备清单

### 服务器环境要求
- **操作系统**: Linux (推荐 Ubuntu 20.04+ 或 CentOS 7+)
- **内存**: 最小 4GB，推荐 8GB+
- **磁盘**: 最小 20GB，推荐 50GB+
- **网络**: 能访问公网（需要拉取Docker镜像和云厂商API）

### 端口规划
```text
8000  - CMDB Web界面
5000  - CMDB API内部端口
23306 - MySQL数据库（映射到宿主机）
6379  - Redis缓存（容器内部）
```

### 检查脚本
```bash
# 1. 检查端口占用
netstat -tuln | grep -E ':(8000|5000|23306|6379)' || echo "端口检查通过"

# 2. 检查Docker安装
docker --version || echo "需要先安装Docker"

# 3. 检查磁盘空间
df -h | grep -E '(Filesystem|/$)' && echo "确保根分区至少20GB空间"

# 4. 检查内存
free -h | grep Mem
```

#### 4.3 配置LDAP到数据库
```bash
# 进入API容器
docker exec -it cmdb-api sh

# 在容器内执行Python
python << 'EOF'
from api.lib.utils import AESCrypto
from api.models.cmdb import CommonData
from api import db
import json

# LDAP配置（根据实际情况修改）
ldap_config = {
    'enable': 1,
    'ldap_server': 'ldap://your-ldap-server.com:389',
    'ldap_domain': 'your-domain.com',
    'ldap_user_dn': 'cn={username},ou=users,dc=your-domain,dc=com'
}

# 加密配置
encrypted = AESCrypto().encrypt(json.dumps(ldap_config))
print(f"加密后的配置: {encrypted}")

# 保存到数据库
common_data = CommonData.query.filter_by(data_type='LDAP').first()
if common_data:
    common_data.data = encrypted
else:
    common_data = CommonData(data_type='LDAP', data=encrypted)
    db.session.add(common_data)

db.session.commit()
print("LDAP配置已保存")
EOF

exit
```

### 第六步：云同步配置（解决问题#7、#8）

#### 6.1 修复内置自动发现规则
```bash
# 进入API容器
docker exec -it cmdb-api sh

# 修复公有云规则的option.en字段
python << 'EOF'
from api.models.cmdb import AutoDiscoveryRule

# 定义云厂商映射
CLOUD_PATCHES = {
    "阿里云": "aliyun",
    "腾讯云": "tencentcloud",
    "华为云": "huaweicloud",
    "AWS": "aws",
}

# 更新规则
for name, en in CLOUD_PATCHES.items():
    rule = AutoDiscoveryRule.query.filter_by(name=name).first()
    if rule:
        option = dict(rule.option or {})
        option["en"] = en
        rule.option = option
        print(f"Updated {name}: {option}")
    else:
        print(f"Warning: {name} rule not found")

from api import db
db.session.commit()
print("云发现规则修复完成")
EOF

exit
```

## 🎯 性能优化建议

### 数据库优化
```bash
# 调整MySQL配置（编辑宿主机上的MySQL配置文件）
cat > /opt/mysql-optimization.cnf << 'EOF'
[mysqld]
# 连接数
max_connections = 500

# InnoDB缓冲池
innodb_buffer_pool_size = 2G

# 查询缓存
query_cache_size = 64M
query_cache_type = 1

# 慢查询日志
slow_query_log = 1
long_query_time = 2

# 二进制日志
max_binlog_size = 100M
expire_logs_days = 7
EOF
```

## 🎓 部署经验总结

### 关键成功因素
1. **环境一致性**：统一开发、测试、生产环境的配置
2. **网络稳定性**：确保DNS解析和容器网络配置正确
3. **配置验证**：逐步验证每个组件，不要一次性启动所有服务
4. **日志监控**：密切关注日志输出，及时发现问题

### 常见错误模式
1. **急于部署**：跳过环境检查直接部署，导致后期频繁返工
2. **忽略DNS**：LDAP配置正确但DNS解析错误，导致认证失败
3. **配置混乱**：SECRET_KEY不一致，导致加密配置无法解密
4. **版本不匹配**：应用版本和数据库表结构不一致

### 最佳实践
1. **分步部署**：按基础->应用->功能的顺序逐步部署
2. **自动化验证**：使用脚本自动检查部署状态
3. **文档记录**：记录所有配置和修改，便于问题排查
4. **备份优先**：部署前先备份，出问题可以快速回滚
