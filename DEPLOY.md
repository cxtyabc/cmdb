## 🎯 核心经验总结

### ⚠️ 关键注意事项

#### 1. 内存配置 (最重要!)
```bash
# ⚠️ 必须配置swap (3.6GB内存服务器)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

#### 2. LDAP配置正确姿势
```bash
# 使用ldapsearch确认格式
ldapsearch -x -H ldap://server:389 -b "dc=domain,dc=com" "(uid=user)" dn

# Web界面使用{}占位符
用户DN: cn={},ou=Devops,ou=Development,dc=domain,dc=com
```

#### 3. 标准化部署路径
```bash
/gaodun/domin/veops-cmdb  # 生产环境
/opt/veops-cmdb           # 备选路径
```

#### 4. 配置文件保护
```bash
# 修改前务必备份
cp docker-compose.yml docker-compose.yml.backup.$(date +%Y%m%d)
```

## 🛠️ 实用工具

### 一键部署脚本
```bash
#!/bin/bash
# 保存为 quick-deploy.sh
echo "=== CMDB快速部署 ==="

# 1. 内存优化
echo "配置swap空间..."
sudo fallocate -l 2G /swapfile 2>/dev/null
sudo chmod 600 /swapfile 2>/dev/null
sudo mkswap /swapfile 2>/dev/null
sudo swapon /swapfile 2>/dev/null

# 2. 启动服务
echo "启动CMDB服务..."
cd /gaodun/domin/veops-cmdb
docker compose up -d

# 3. 验证服务
echo "验证服务状态..."
sleep 30
curl -s -o /dev/null -w 'HTTP状态码: %{http_code}\n' http://localhost:8000/
docker ps --format 'table {{.Names}}\t{{.Status}}' | grep cmdb
```

### 健康检查脚本
```bash
#!/bin/bash
# 保存为 health-check.sh
echo "=== CMDB健康检查 ==="

# 容器状态
echo "容器状态:"
docker ps --format 'table {{.Names}}\t{{.Status}}' | grep cmdb

# 资源使用
echo -e "\n资源使用:"
free -h | head -2
df -h | grep -E '(Filesystem|/$)'

# 服务测试
echo -e "\n服务测试:"
curl -s -o /dev/null -w "  Web界面: %{http_code}\n" http://localhost:8000/
docker exec cmdb-api ping -c 1 cmdb-db > /dev/null 2>&1 && echo "  数据库连接: 正常" || echo "  数据库连接: 失败"
```
