## 🚨 紧急故障处理

### 🔴 严重问题 (立即处理)

#### 问题：所有服务无法启动
**症状**: `docker compose up -d` 失败
**快速修复**:
```bash
# 1. 检查Docker服务
sudo systemctl status docker
sudo systemctl restart docker

# 2. 检查端口占用
sudo netstat -tuln | grep -E ':(8000|23306)'
sudo lsof -ti:8000 | xargs sudo kill -9

# 3. 检查磁盘空间
df -h | grep -E '(Filesystem|/$)' | awk '{print $5}' | grep 9[0-9]%

# 4. 检查内存
free -h | grep Mem | awk '{print $4}' | grep [0-9][0-9][0-9]M
```

#### 问题：内存不足导致容器频繁重启
**症状**: Exit code 137, 容器不断重启
**快速修复**:
```bash
# 立即创建swap文件
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 验证
free -h  # 应显示2GB swap
```

#### 问题：数据库连接完全失败
**症状**: API返回200但所有操作报数据库错误
**快速修复**:
```bash
# 1. 检查数据库容器
docker ps | grep cmdb-db

# 2. 强制重启数据库
docker compose restart cmdb-db

# 3. 检查网络
docker exec cmdb-api ping -c 2 cmdb-db

# 4. 检查环境变量
docker exec cmdb-api env | grep MYSQL_HOST
```

## 🟡 常见问题 (10分钟内解决)

### LDAP相关问题

#### LDAP用户无法登录
**可能原因**: DNS解析错误 | 用户DN格式错误 | 占位符转义
**诊断步骤**:
```bash
# 1. 检查DNS解析
docker exec cmdb-api ping your-ldap-server.com

# 2. 测试LDAP连接
docker exec cmdb-api python3 << 'EOF'
from ldap3 import Server, Connection
server = Server('your-ldap-server.com')
try:
    conn = Connection(server, 'cn=testuser,ou=users,dc=domain,dc=com', 'password')
    print(f"LDAP连接: {'成功' if conn.bind() else '失败'}")
except Exception as e:
    print(f"连接失败: {str(e)[:100]}")
EOF

# 3. 检查配置格式
# Web界面 → 设置 → 认证 → LDAP
# 确认使用 {} 或 {0} 占位符
```

**解决方案**:
```bash
# 方案A：修正用户DN格式
# 错误：cn={username},dc=domain,dc=com
# 正确：cn={},ou=Devops,ou=Development,dc=domain,dc=com

# 方案B：添加DNS映射
# 编辑docker-compose.yml:
extra_hosts:
  - "ldap.server.com:172.29.255.244"

# 方案C：使用ldapsearch确认格式
ldapsearch -x -H ldap://server:389 -b "dc=domain,dc=com" "(uid=user)" dn
```

#### LDAP测试通过但实际登录失败
**原因**: Web界面占位符被转义
**解决**:
- 将`{username}`改为`{}`或`{0}`
- 重新保存配置
- 测试实际登录

### 网络相关问题

#### 容器间无法通信
**症状**: API无法连接数据库，ping不通
**诊断**:
```bash
# 检查网络
docker network ls
docker network inspect veops_cmdb_network

# 测试连接
docker exec cmdb-api ping cmdb-db
docker exec cmdb-api ping cmdb-cache
```

**解决方案**:
```bash
# 重建网络
docker compose down
docker network prune -f
docker compose up -d
```

#### 端口冲突
**症状**: 服务无法启动，提示端口已被使用
**解决**:
```bash
# 查找占用端口的进程
sudo lsof -ti:8000
sudo netstat -tuln | grep 8000

# 停止占用进程
sudo lsof -ti:8000 | xargs sudo kill -9

# 或修改端口映射
# 在docker-compose.yml中修改：8000:80 → 8001:80
```

### 配置相关问题

#### 环境变量丢失导致连接127.0.0.1
**症状**: API报错连接到127.0.0.1而不是cmdb-db
**诊断**:
```bash
docker exec cmdb-api env | grep MYSQL_HOST
# 如果为空，说明环境变量丢失
```

**解决**:
```bash
# 检查docker-compose.yml
grep -A 20 "cmdb-api:" docker-compose.yml | grep MYSQL_HOST

# 重新启动服务
docker compose restart cmdb-api
```

#### 配置文件损坏
**症状**: docker compose parse error
**解决**:
```bash
# 从备份恢复
cp docker-compose.yml.backup docker-compose.yml

# 或从版本控制恢复
git checkout docker-compose.yml

# 验证语法
docker compose config
```

## 🚀 性能优化

### 内存优化
```bash
# 检查内存压力
vm.stat 1 100 | grep pressure
# 值 > 0 表示内存压力

# 清理缓存
sync && echo 3 > /proc/sys/vm/drop_caches

# 检查OOM情况
dmesg | grep -i "out of memory" | tail -10
```

### 磁盘清理
```bash
# Docker清理
docker system prune -a -f --volumes

# 日志清理
docker logs cmdb-api 2>&1 | tail -1000 > /tmp/cmdb-api.log
docker logs cmdb-api 2>&1 | wc -c  # 检查日志大小

# 系统日志清理
sudo journalctl --vacuum-time=7d
sudo logrotate -f /etc/logrotate.conf
```


### 常见错误码
- **Exit code 137**: 内存不足，被OOM Killer终止
- **Exit code 125**: 容器命令执行失败
- **Exit code 126**: 容器命令权限问题
- **Exit code 127**: 容器内命令不存在
